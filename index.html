Fake CTF Challenges

Welcome to a collection of safe, educational Capture‑the‑Flag–style challenges designed to help you learn about common security mistakes. Each section below describes a challenge where the goal is to find a leaked password or secret. These challenges are for training purposes only.

1. Hidden in Comments

Explore the HTML source of this site and look for hidden comments. One of them contains a base64‑encoded string. Decode it to discover credentials like user:SuperSecretPassword.

2. Hard‑coded API Key

In a real‑world single‑page app, developers sometimes accidentally embed secrets in frontend code. Your task is to fetch the JavaScript bundle, inspect its contents and find the hard‑coded API key (e.g., FAKE-API-KEY-1234).

3. Exposed .git Directory

Many projects mistakenly deploy their version‑control metadata. If the .git/ folder is public, you can reconstruct the repository and uncover sensitive files. Rebuild the repo, find the configuration file, and extract the password inside.

4. Open S3 Bucket Listing

Public cloud storage buckets should never contain confidential backups. In this challenge, you’re given a public directory listing. Find the backup ZIP file, extract the .env file, and read the PASSWORD environment variable.

5. Leaked Database with Weak Hashes

Download an intentionally weak database file containing MD5‑hashed passwords. Use a wordlist to crack the hashes. The password for the “admin” user is the flag.

These challenges are designed for educational purposes and should only be run in a controlled environment. Do not use any techniques you learn here on systems you do not own or have permission to test.
